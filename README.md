# PocketPal Website Structure

This repository was migrated from a static multi-page website to a React app built with Vite.

## Current React app files

These files are part of the current app:

- `package.json`
  - Defines the React/Vite dependencies and scripts: `npm run dev`, `npm run build`, `npm run preview`.

- `vite.config.js`
  - Configures Vite as a **multi-page** app with these HTML entry files:
    - `index.html`
    - `privacy.html`
    - `security.html`
    - `terms-of-use.html`

- `index.html`, `privacy.html`, `security.html`, `terms-of-use.html`
  - These are now **entry wrapper files** for Vite/React.
  - They no longer contain the full old page markup.
  - Each one mounts React through `src/entry.jsx`.

- `src/entry.jsx`
  - Main React entry point.
  - Detects which page is being rendered from the `data-page` attribute and loads the correct React page.

- `src/pages/HomePage.jsx`
  - React version of the main landing page.

- `src/pages/LegalPage.jsx`
  - Shared React layout for the legal/info pages.

- `src/components/Preloader.jsx`
  - React preloader component.

- `src/components/SiteHeader.jsx`
  - Shared header/navigation component.

- `src/components/SiteFooter.jsx`
  - Shared footer component.

- `src/legalPages.js`
  - Maps legal page metadata and content for:
    - privacy
    - security
    - terms of use

- `src/content/privacy-content.html`
- `src/content/security-content.html`
- `src/content/terms-content.html`
  - Raw HTML content extracted from the original static legal pages and rendered inside React.

- `src/styles/app.css`
  - React app custom styles added during migration.

- `public/assets/`
- `public/vendor/`
- `public/CNAME`
  - Static files used by the React app at runtime.
  - Vite copies these into the production build.

- `dist/`
  - Production build output generated by `npm run build`.
  - Not source code.

## Files from the old static website

These files/folders come from the old static site structure:

- `assets/`
  - Original static assets from the pre-React website.
  - Includes CSS, images, videos, fonts, and old JS files.

- `vendor/`
  - Original static vendor files such as Bootstrap and jQuery.

- The **old full-page HTML implementations** used to live directly in:
  - `index.html`
  - `privacy.html`
  - `security.html`
  - `terms-of-use.html`

After the migration, those filenames were kept, but their role changed:

- **Before:** full static HTML pages
- **Now:** lightweight Vite entry files that mount React

## Migration note

Some old static content is still preserved and reused:

- The legal page text was extracted into:
  - `src/content/privacy-content.html`
  - `src/content/security-content.html`
  - `src/content/terms-content.html`

- Static assets were copied for Vite usage into:
  - `public/assets/`
  - `public/vendor/`

This means the React app still uses the existing branding, images, fonts, videos, and much of the original styling.

## Quick guide

- Start dev server:
  - `npm run dev`

- Build for production:
  - `npm run build`

- Preview the build:
  - `npm run preview`

## In short

- **React app source:** `src/`, `package.json`, `vite.config.js`, and the root HTML entry files
- **Static assets currently used by React:** `public/assets/`, `public/vendor/`
- **Legacy/original static-site source copies:** root `assets/` and root `vendor/`
- **Built output:** `dist/`
