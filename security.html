Security

Our Commitment to Data Protection

At PocketPal, we take security seriously. This page outlines the security measures we implement to protect your data and ensure secure access to our Services.

1. Security Architecture

Our security architecture is designed with multiple layers of protection:

  • • Server-Side Validation: All role assignments and permissions are validated on the server, not the client
  • • JWT Token Verification: Authentication tokens are verified with Supabase before granting access
  • • Role-Based Access Control: Granular permissions ensure users only access features appropriate for their role
  • • Database Security: Row Level Security (RLS) policies protect sensitive data

2. Authentication & Authorization

We use secure authentication methods to protect your account:

  • • Magic Link Authentication: Passwordless login via secure email links
  • • Email Verification: All accounts require verified email addresses
  • • Role Whitelisting: Admin roles are restricted to pre-approved email addresses
  • • Session Management: Secure session handling with automatic timeout capabilities

3. Data Protection

Your data is protected through multiple security measures:

  • • Encryption: Data is encrypted in transit (TLS/SSL) and at rest
  • • Access Controls: Role-based permissions limit who can access what data
  • • Database Security: Row Level Security policies ensure users can only access their own data
  • • Secure Storage: Data is stored in secure cloud infrastructure with industry-standard protections

4. Security Features

Role Tampering Prevention

Client-side validation detects unauthorized role modifications and automatically logs out affected users.

Permission-Based Access

Each role has specific permissions. Admin features check permissions before rendering, preventing unauthorized access.

Security Monitoring

Continuous logging and monitoring help us detect and respond to security incidents quickly.

5. Security Best Practices

We follow industry best practices to maintain security:

  • • Principle of Least Privilege: Users start with minimal permissions and only receive additional access when necessary
  • • Regular Security Audits: We regularly review security events and access logs
  • • Environment Variables: Sensitive configuration is stored securely in environment variables
  • • Secure Development: Code follows security best practices and is regularly reviewed

6. Your Role in Security

You can help keep your account secure by:

  • • Keeping your email account secure (since we use email-based authentication)
  • • Not sharing your account credentials with others
  • • Logging out when using shared devices
  • • Reporting any suspicious activity to pocketpalrevolution@gmail.com

7. Security Incident Response

If we become aware of a security breach that affects your personal information, we will notify you and any affected institutions as required by law. We have procedures in place to respond to security incidents promptly and effectively.

8. Future Security Enhancements

We are continuously working to improve our security. Planned enhancements include:

  • • Multi-factor authentication (MFA) for admin accounts
  • • Enhanced session management with automatic timeouts
  • • Comprehensive audit logging for all admin actions
  • • IP whitelisting for administrative access (optional)

9. Contact Us

If you have security concerns or questions, please contact us at pocketpalrevolution@gmail.com. For more information about how we protect your data, please see our Privacy Notice.
